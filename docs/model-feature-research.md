# 予測モデル特徴量調査と採用方針

## 原則

考えられる要素は特徴量候補へ登録する。ただし候補登録と本番採用を分け、時系列外検証で確率精度・校正・期待値が改善した特徴量群だけを採用する。対象レースの結果、払戻、締切後に得た情報は入力へ入れない。

## 特徴量群

1. レース条件: 競馬場、コース、芝・ダート・障害、距離、回り、クラス、頭数、季節、発走時間。
2. 天候・馬場: 天候、良・稍重・重・不良、含水率、クッション値、同日中の変化。
3. コース形状: 直線長、高低差、坂、コーナー数、スタート位置、最初のコーナーまでの距離。
4. 馬の能力: 近走着順、時計、上がり、通過順、位置取り変化、休養日数、成長・衰退、安定度。
5. 条件適性: 競馬場、距離、芝ダ、馬場状態、右左回り別の過去性能。
6. 展開: 脚質、先行馬密度、想定ペース、末脚の持続、枠とコース形状の相互作用。
7. 馬体・斤量: 性齢、馬体重、増減、斤量、斤量体重比、前走からの変化。
8. 人: 騎手、調教師、騎手と馬、騎手と調教師のローリング成績。競馬場・距離・馬場別に分解する。
9. 血統・調教: 父・母父系統、産駒傾向、追切時計、調教コース、併せ内容、ローテーション、輸送。
10. 市場: 全券種の発走前オッズ、時系列変動、票数、過剰人気、券種間整合性。
11. 相手関係: メンバー能力分布、上位差、混戦度、クラス変動、逃げ先行構成。
12. 気象実測・直前変更: 気温、雨、風向風速、湿度、取消、除外、騎手変更、馬体重発表。

## 現在の無料データ範囲

現行DBでレース条件、天候・馬場区分、馬の過去走、条件適性、展開履歴、馬体・斤量、騎手・調教師、相手関係、単複市場を生成できる。コース形状、含水率・クッション値、血統、調教、外部気象、全券種発走前オッズは追加取得が必要。

JRAは馬場状態を4区分で公表し、含水率と馬場状態区分は同一ではないと説明している。したがって両者を別特徴量とし、含水率だけから馬場状態を再構成しない。

- JRA 馬場状態・クッション値: https://www.jra.go.jp/keiba/baba/kaisetsu/index.html
- JRA 過去の含水率・クッション値: https://www.jra.go.jp/keiba/baba/archive/
- JRA 競馬場コース紹介（例・東京）: https://www.jra.go.jp/facilities/race/tokyo/course/index.html

## モデル構成

- 基本能力モデル: レース内多項確率を出すランキングモデル。
- 順位分解: Plackett-Luce型で2着・3着までの条件付き確率を生成。
- 市場融合: 基本能力確率と発走前市場確率を第2段モデルで統合。
- 券種校正: 単勝、複勝、馬連、ワイド、馬単、3連複、3連単ごとに校正。
- 不確実性: 開催日単位のblock bootstrapで安全側確率・期待回収率を作る。

精度指標だけでモデルを選ばず、log loss、Brier score、ECE、reliability curveを優先する。スポーツベッティングでは確率校正が収益判断に重要という検証結果とも整合する。

- Machine learning for sports betting: should model selection be based on accuracy or calibration?: https://arxiv.org/abs/2303.06021
- William Benter, Computer Based Horse Race Handicapping and Wagering Systems: https://gwern.net/doc/statistics/decision/1994-benter.pdf

## 特徴量採用ゲート

各特徴量群を除いたモデルとのアブレーション比較をwalk-forwardの各foldで実行する。log loss改善が規定値未満、校正悪化、期間依存が強い、観測時刻を証明できない、欠損率が高い特徴量は採用しない。回収率だけが偶然改善した特徴量も採用しない。

実装上は29特徴量を、レース条件、馬体・負担重量、近走成績、馬場・距離適性、騎手・調教師、相手関係の6群に分ける。各foldの学習期間で全群モデルと1群除外モデルを個別学習し、校正期間で温度を再推定して比較する。全群モデルのlog loss改善が0.0001以上、ECE悪化が0.001以下、最大校正bin誤差が0.075以下の場合にそのfoldを合格とし、全foldの60%以上で合格した群だけを最終モデルへ採用する。採用群がゼロの場合は全特徴量モデルをベンチマークとして生成するが、研究合格にはしない。
