# 無課金JRAデータ蓄積基盤

## 目的

JRA公式の「過去レース結果検索」から、1996年1月以降の中央競馬結果をローカルSQLiteへ蓄積する。取得不能・解析不能・欠損を成功扱いしない。

## 取得経路

1. JRA公式の過去レース結果検索画面から年月トークンを取得する。
2. 年月ページからJRAが返した開催リンクだけを抽出する。
3. 開催ページからJRAが返した全レースリンクを抽出する。
4. レースページを取得し、出走馬、騎手、調教師、着順、馬体重、人気、払戻を正規化する。

URLを総当たりで推測しない。アクセス間隔を設け、HTTP失敗は指数バックオフで最大4回まで再試行する。

## 保存と監査

- 原本HTMLはShift_JISからUTF-8へ正規化し、gzipでローカル保存する。
- 全原本にSHA-256を記録する。
- 月、開催、レース、出走馬、払戻を別テーブルに保存する。
- レース単位のDB更新はトランザクションで行う。
- 月の公式リンク件数とDB件数を照合する。
- 勝馬または払戻がないレースが1件でもあれば月全体を失敗にする。
- レース名、距離、馬場種別、発走時刻、馬ID、馬番、騎手ID、調教師IDを必須検査する。
- 再実行は保存済み原本を再利用し、主キーと一意制約で重複を防ぐ。
- `audit`は全gzipを展開し、SHA-256、参照切れ、品質検査を再確認する。
- `complete_*`ビューは月次品質ゲート合格済みの行だけを返す。モデルと画面は物理テーブルを直接参照しない。
- 公式IDリンクがない外国人騎手・地方所属調教師は`NAME:`付き代替IDで明示し、公式IDと区別する。
- 公式掲載自体がない人気、取消・競走中止等の時計や馬体重はNULLのまま保持し、`complete_entry_quality`で特徴量ごとの利用可否を返す。

## 実行

```powershell
$node = "$env:USERPROFILE\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe"
& $node scripts\jra-free-db.mjs init --from 1996-01
& $node scripts\jra-free-db.mjs run --limit 1 --delay 1500
& $node scripts\jra-free-db.mjs audit
& $node scripts\jra-free-db.mjs status
```

DBと原本は`data/jra-free-private/`に保存し、Git管理・公開配信から除外する。

## 期待値検証の制約

JRAの無料過去結果には全馬の人気はあるが、全馬の確定単勝オッズ時系列はない。勝馬の払戻から勝馬オッズだけを復元してEV検証すると選択バイアスになる。

そのため次を必須とする。

- 過去結果は能力確率モデルの時系列学習と校正に使う。
- EVは予測時点の実測オッズだけで計算する。
- 今後、全馬オッズのスナップショットを時系列蓄積する。
- 十分なオッズ履歴がない期間はEVバックテストを`insufficient`にする。
- 校正、時系列分割、オッズ鮮度、データ完全性の全ゲート合格まで買い目を生成しない。

## 公式情報上の注意

JRAは1986年以降の結果を掲載しているが、2000年以前は不完全なデータや現在と異なる表示ルールがあると明記している。このため旧形式専用パーサーと月単位の完全性検査を維持する。

- 過去結果掲載範囲: https://www.jra.go.jp/faq/pop02/1_6.html
- JRAホームページ利用案内: https://www.jra.go.jp/use/
- robots.txt: https://www.jra.go.jp/robots.txt

原本HTMLとローカルDBは公開しない。公開画面には検証済みの限定的な事実データ、集計値、派生予測だけを載せ、画像・映像・記事本文は転載しない。
