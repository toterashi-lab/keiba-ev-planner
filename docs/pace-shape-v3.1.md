# ペース形状特徴量 v3.1

## 目的

JRA結果ページから取得済みの過去通過順位を、対象レースの結果ではなく各馬の過去レース履歴として期待値モデルへ入力する。

## 実装

- 最初のコーナー位置、最終コーナー位置、位置取り上昇量、逃げ・先行頻度を出走頭数で正規化する。
- 直近3走の先行位置、最終コーナー位置、位置取り上昇量を生成する。
- 出走馬全体の想定先行密度と、各馬の先行頻度・位置取り上昇量との相互作用を生成する。
- 同一発走時刻の別レース結果も使用せず、発走時刻単位で一括更新する。
- 条件別・騎手別・調教師別マップには通過順位配列を複製せず、馬本体の履歴だけに保持する。
- `pace_shape`特徴量群は強制採用せず、各walk-forward foldのgroup ablationを通過した場合だけ採用する。

## 実データ事前検証

- 対象: 45,284レース、630,593学習行
- 採用特徴量群: `race_context`、`body_load`、`horse_form`、`pace_shape`
- log loss: 2.15752
- uniform log loss: 2.59022
- ECE: 0.00329
- 最大校正bin誤差: 0.01348
- 全7券種確率校正: 合格
- 推定全期間RSS: 7,638MB
- 推定全期間所要時間: 59.6分

この事前検証中もバックフィルDBは増加していたため、鮮度ゲートは意図どおり不合格となった。30年分完了後に固定された最新DBで事前検証とリソースゲートを再実行する。
