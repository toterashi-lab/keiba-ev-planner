# UIフレームワーク v4

## 画面の役割

競馬開催日ごとに、期待値の高いレースと買い目を短時間で比較し、根拠・検証結果・データ品質へ段階的に掘り下げる分析画面。

## 参照デザインから抽出した構造

- 上部はブランドより現在の主要コンテンツを優先する。
- 主役画像と短い説明で、最初に見るべき分析対象を明確にする。
- 関連コンテンツを横方向の列にまとめ、縦スクロール量を抑える。
- 選択中、ホバー、フォーカス状態を背景・境界線・拡大率で明確にする。
- 詳細情報は選択後に展開し、一覧画面へ常時詰め込みすぎない。

## 競馬分析向けの再構成

- 映像作品のカード列を、開催日・競馬場・券種・買い方・レース期待値の列へ置き換える。
- 作品メタデータを、発走時刻・券種・買い目・安全側期待回収率・購入可否へ置き換える。
- 自動再生や巨大プレビューは使わず、高密度な比較と誤操作防止を優先する。
- 購入不可・外部検証不合格は、カード選択やスクロール後も隠さない。

## CSSシステム

- 背景: 黒ではなく緑味を抑えたチャコール。
- 主アクセント: エメラルド。選択、進捗、データ正常を表す。
- 補助アクセント: ゴールド。順位、重要数値、主操作に限定する。
- 情報アクセント: ブルー。研究資料、モデル情報、補助操作に使う。
- 警告: 赤。購入不可、外部検証不合格、欠損だけに使う。
- 角丸は最大6px。分析カード、モーダル、繰り返し要素に限定する。
- 表、タブ、レールは固定寸法と横スクロールを使い、動的データでレイアウトを移動させない。

## OSS・依存方針

- 現在のHTML/CSS/JavaScript構成を維持し、外部CDNや有料UIライブラリを追加しない。
- GitHub Pagesでそのまま配信できること、取得処理停止中でも画面が開けることを優先する。
- アイコンライブラリが導入されるまでは、既存の明確なテキスト操作を維持する。
- 将来コンポーネント分割が必要になった場合のみ、ビルド不要のWeb Componentsを第一候補とする。

## モバイル

- 上部ナビ、開催日、競馬場、券種、買い方、レース番号は横スクロールとする。
- 主役領域の高さを抑え、次のランキング列が最初の画面内に見えるようにする。
- ランキングカードは画面幅の約76%に固定し、次カードの一部を見せて横操作を示す。
- 表は縮小せず横スクロールし、数値と見出しの対応を維持する。
