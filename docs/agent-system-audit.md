# 5エージェント基盤監査

更新日: 2026-07-22

## 現在の実装状況

- 静的HTML/CSS/JavaScriptとNode.jsのSQLite生成スクリプトで構成する。
- 既存のライブ候補生成、発走前オッズの時刻検査、順位確率モデル、期待値候補の記録は再利用する。
- 新規の`agent_predictions`系テーブルは、5エージェントの不変予想、馬別確率、買い目、精算、採点、学習履歴を保存する。

## 不完全だった計算

- 旧AIカードは同じ確率列を異なる並べ替えで表示しており、独立した重み・閾値・学習履歴を持たなかった。
- 旧AI別成績は各AI本命の単勝100円を仮定しており、保存済み推奨買い目を精算していなかった。
- BOX・フォーメーションの候補が、購入推奨と候補ランキングで区別されていなかった。

## モック・固定値

- 画面の旧`forecast-policy`は、全レースを表示するための参考買い目を作成していた。本番推奨のフォールバックから除外した。
- テスト用fixtureは隔離用途だけに残し、実データであるように表示しない。

## 学習ループの欠落と対策

- 旧実装にはエージェント別の時系列更新、校正モデル、ロールバック可能なモデル版がなかった。
- `training_runs`、`model_versions`、`calibration_models`、`feature_weights`、`learning_events`を追加し、最小サンプル数、更新幅上限、正則化を適用する。

## データリークの危険

- 締切後オッズ、確定着順、払戻、確定人気を入力へ混入させない。
- 公開予想は追記専用。修正は別バージョンとして保存する。
- 時系列学習はウォークフォワード分割のみを許可する。

## 再利用する既存機能

- `finish-order-probabilities.mjs`のPlackett-Luce型順位確率。
- ライブオッズの発走前時刻判定とバッチ出典。
- 既存の結果・払戻データ、モバイル用カード、結果画面、テーマ切替。

## 削除・統合候補

- 旧5カードの同型スコアリングは新エージェント出力へ統合する。
- `forecast-policy`は荒れ具合表示だけに限定し、買い目作成には使わない。

## 本番化に必要な追加実装

- 結果バックフィルと履歴オッズの全件完了。
- 時系列学習、券種別校正、外部検証、ROI信頼区間、最大ドローダウンの合格。
- 総合エージェントの不変スナップショット保存。
